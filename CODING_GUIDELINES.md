# MultiConnect 编码准则

> 适用于主代理与所有并行开发代理。出现冲突时，以安全、数据完整性和模块边界为先。

## 1. 基本原则

- 当前项目仍处于工程基线阶段，不得把演示数据描述为真实平台数据。
- 优先实现小而完整、可以验证的闭环，避免一次加入过多依赖。
- 平台接入、翻译、远程控制分别通过适配器隔离，不在 UI 中写平台专用逻辑。
- 不绕过第三方平台验证、风控或使用条款。
- 不覆盖或回滚其他代理的改动；发现冲突时适配现有结构。

## 2. 模块边界

- `src/`：React 展示与交互，不保存 Session、API Key 或配对密钥。
- `src-tauri/`：本地可信边界，负责密钥、持久化、任务队列和适配器监管。
- `server/`：我们自己的 Web 控制台后端，负责用户、设备配对、命令路由和审计。
- 平台 Session 只能留在客户端，禁止上传到 `server/`。
- Web 后端只能向已配对设备发送受权限控制的命令。

## 3. TypeScript

- 开启严格类型，禁止无说明的 `any` 和 `@ts-ignore`。
- 对外部输入先校验再使用；网络错误必须转为稳定错误码。
- React 组件保持单一职责，复杂状态进入 Hook 或 service。
- 不在组件中散落 `fetch()`；统一进入 `src/lib/` 或 feature service。
- 用户可见操作必须包含加载、成功、空状态和失败反馈。

## 4. Rust

- 业务路径禁止 `unwrap()` 和 `expect()`；应用最外层启动失败除外。
- Tauri command 只做参数校验和服务调用，不承载复杂业务逻辑。
- 敏感类型不得输出完整 `Debug` 信息。
- 加密必须使用认证加密和随机 nonce，禁止自制加密算法。
- 所有状态变化应可测试，并返回稳定错误码。

## 5. API 与实时通信

- REST 路径使用 `/api/v1` 版本前缀；健康检查可使用 `/health`。
- WebSocket 只接受已认证、已配对设备；消息包含版本、ID、类型和时间。
- 命令必须有幂等 ID、超时和确认结果。
- 日志不得包含 Session、Token、API Key、完整手机号或消息正文。
- 开发环境允许 localhost HTTP；非本地地址必须使用 HTTPS/WSS。

## 6. 翻译

- 接收消息保留平台原文，译文作为派生数据单独展示。
- 发送消息必须先显示译文预览，未经用户确认不得自动发送。
- 翻译失败不应阻断查看原消息。
- Provider、源语言、目标语言、术语版本和文本哈希共同构成缓存键。
- React 前端不得持有真实翻译服务密钥。

## 7. 测试与验收

- 新增纯逻辑必须提供单元测试或至少可执行的验证入口。
- 不使用个人账号、生产令牌或真实联系人作为测试数据。
- 提交前运行适用的 build、typecheck 和测试命令。
- 代理完成任务时必须列出修改文件、验证命令和未解决风险。

## 8. 并行协作

- 每个代理只修改分配的文件范围。
- 不修改根配置文件，除非任务明确授权。
- 如果必须跨边界改动，先在结果中提出，不擅自扩大范围。
- 不假设其他代理未在工作；读取最新文件后再编辑。
