# MultiConnect 项目当前进度总结

更新时间：2026-06-21

## 当前整体进度

项目已经从“能打开 WhatsApp Web”推进到“可多账号测试 + 可翻译 + 可本地 Web 控制台查看状态”的阶段。

当前最核心的主线有三条：

1. 桌面客户端：多 WhatsApp 账号独立 Session、翻译、账号管理。
2. 本地控制服务：HTTP + WSS，负责设备注册、状态命令、控制台 API。
3. Web 控制台：查看设备和账号状态，并触发账号检测。

## 已完成模块

### 1. 多账号 WhatsApp 客户端

- 每个 WhatsApp 账号使用独立 Session。
- 标签固定，不会因为点击账号跳来跳去。
- 顶部账号过多时有“更多”折叠和快速切换。
- 左侧只保留账号汇总图表式入口，减少 50 个账号时的拥挤。
- 账号管理页支持搜索、筛选、备注编辑、置顶、关闭标签、重新登录、删除入口。
- 当前已经测试过约 10-20 个账号级别的界面表现。

### 2. 翻译功能

- OpenAI 翻译已经接入。
- 支持自然口语化翻译。
- 支持翻译语气设置方向，后续可扩展为“亚洲口气 / 欧洲口气 / 商务 / 朋友式”等选项。
- 发送前翻译预览已修正为：
  - 第一次点击发送按钮或按 Enter：只生成/替换译文。
  - 第二次点击发送或 Enter：才发送。
- 翻译失败时不会直接把中文原文发出去。
- 接收消息支持翻译按钮和自动翻译。
- 接收翻译结果已持久化缓存。
- 设置页已有缓存管理和翻译日志。

### 3. 性能和缓存

- 翻译缓存本身占用很小，当前测试量级只有 KB 级。
- 真正吃资源的是多开 WebView/WhatsApp 页面，不是翻译缓存。
- 当前客户端相比完整浏览器多开更轻，但如果 50 个账号同时活跃 WebView，仍然会吃内存，需要后续做休眠/冻结策略。

### 4. 本地控制服务

- 服务目录：`server`
- 默认地址：`127.0.0.1:8000`
- 健康检查：`/health`
- Web 控制台：`/console`
- 当前设备注册是内存模式：
  - 优点：开发简单。
  - 缺点：服务重启后设备注册和命令状态会丢失。

### 5. Web 控制台

- 第一版控制台已可用。
- 可以看到：
  - 服务是否在线。
  - 设备数量。
  - 在线设备数量。
  - 账号总数。
  - 设备状态。
  - 账号列表。
  - 账号在线/离线/待登录/异常。
- 支持账号搜索和筛选。
- 支持单账号“检测”，让客户端刷新该账号状态并回报。

## 当前风险点

### 1. Web 控制台状态仍是本地快照

账号是否在线、是否异常，依赖客户端打开或检测过对应账号。没有打开过的账号可能只显示离线或无补充摘要。

### 2. 封号/异常检测需要继续积累规则

WhatsApp 没有给我们官方封号状态 API。当前只能根据 Web 页面出现的文本、扫码页、错误页、限制提示做判断。

### 3. 批量检测还没做队列

现在是单账号检测。下一步如果做批量检测，必须限制并发，否则 50 个账号同时打开检测会吃资源，也可能让页面乱跳。

### 4. 控制台还不是正式远程运维系统

现在适合本机或内网开发测试。要真正远程使用，还需要：

- 登录鉴权。
- TLS。
- 操作审计。
- 持久化数据库。
- 权限区分。

## 推荐下一步路线

### P0：Web 控制台批量安全检测

目标：可以一次选择多个账号检测，但不会让客户端乱跳或卡死。

建议设计：

- 账号卡片支持多选。
- 顶部出现“检测选中账号”。
- 检测队列默认并发 1。
- 每个账号显示检测中/成功/失败/耗时。
- 可随时停止剩余队列。

### P1：控制服务持久化

目标：服务重启后仍能看到上次设备和账号状态。

建议先用本地 JSON 或 SQLite，后面再升级。

### P2：Web 控制台安全登录

目标：后续能给别人访问，但不泄露本机控制能力。

建议：

- 管理员登录。
- Control API Key 不再只靠浏览器本地保存。
- 操作日志。

### P3：账号异常规则中心

目标：更稳定识别封号、受限、扫码、掉线。

建议：

- 把检测规则集中到一个文件。
- 支持多语言关键词。
- 在控制台显示命中的规则和截图/摘要。

## 给下一位开发者的提醒

- 不要把 OpenAI Key 写进代码或安装包。
- 不要让 Web 控制台拿 WhatsApp Cookie、Session 或聊天正文。
- 不要批量同时打开几十个 WebView。
- 改 WebView 注入脚本后，记得提示用户关闭旧标签重开。
- `cargo fmt` 可能格式化不相关 Rust 文件，提交前检查 diff。
- 中文文档如果在 PowerShell 里显示乱码，优先确认编码，不要误以为内容坏了。
